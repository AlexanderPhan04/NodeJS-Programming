Yêu cầu chung
Xây dựng một RESTful API cho hệ thống Blog cá nhân, nơi người dùng có thể:

Đăng ký / đăng nhập
Viết, sửa, xóa bài viết của chính mình
Xem tất cả bài viết (của mọi người)
Upload ảnh đại diện cho bài viết
Tìm kiếm / lọc bài viết

Dữ liệu lưu bằng MongoDB (khuyến khích) hoặc file JSON nếu chưa quen DB.
Bắt buộc sử dụng JWT cho authentication.
Các entity chính

User
username (unique)
email (unique)
password (hashed bằng bcrypt)
role: "user" hoặc "admin" (admin có quyền xóa bài viết của người khác)

Post (bài viết)
title (string)
content (string hoặc markdown)
author: ObjectId ref User
createdAt, updatedAt
imageUrl (string – link ảnh upload)
tags: array string (ví dụ: ["Node.js", "Express", "Web"])

Yêu cầu kỹ thuật & nghiệp vụ (bắt buộc)

Authentication: Dùng JWT (jsonwebtoken) + middleware kiểm tra token
Authorization: Chỉ chủ bài (author) hoặc admin mới sửa/xóa được bài của mình
Password: Hash bằng bcrypt
Upload ảnh: Sử dụng multer để upload file ảnh bài viết (lưu vào thư mục public/uploads, trả về URL)
Validation: Dùng Joi hoặc express-validator để validate input (title/content không rỗng, email hợp lệ,…)
Phân trang: Hỗ trợ query ?page=1&limit=10 cho GET /posts
Tìm kiếm: Hỗ trợ ?q=keyword (tìm trong title + content) và ?tag=Node.js
Error handling: Middleware xử lý lỗi toàn cục, trả JSON { error: "message" } + status code phù hợp
